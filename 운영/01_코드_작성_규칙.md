# Unity C# 코드 작성 규칙

이 문서는 프로젝트 내 C# 스크립트 작성 시 따라야 할 규칙의 단일 소스다.
모르겠으면 이 문서를 먼저 확인하고, 그래도 모르면 팀에 물어봐도 됨.

---

## 0) TL;DR (3줄)
1) **소모/증가 코스트**는 `BaseCurrency` 상속 후 `SaveKey`·`startingAmount`만 재정의
2) **열거형**은 `PascalCase`로 선언, 멤버는 의미 있는 단어로 명명 + `= 0` None 포함
3) **변수명**은 용도를 즉시 알 수 있도록 — 약어·축약어 금지

---

## 1) 코스트(Currency) 클래스 작성 규칙

### 기본 원칙
소모되는 코스트 (스택, HP, 영혼석 등) 는 모두 `BaseCurrency`를 상속받아 구현한다.  
`BaseCurrency`는 소모·증가·저장·로드의 기본 동작을 제공하는 추상 클래스이므로,  
최소/최대값이나 UI 연동 같은 세부 사항은 각 자식 클래스에서 추가/수정한다.

### 반드시 재정의해야 하는 항목

| 항목 | 종류 | 설명 | SoulstoneManager 예시 |
|---|---|---|---|
| `SaveKey` | `abstract` 프로퍼티 | PlayerPrefs 저장 키. **코스트 이름**으로 작성 | `"SoulStone"` |
| `startingAmount` | `virtual` 멤버 변수 | 게임 시작 시 초기값 | `10` |

```csharp
// ✅ 올바른 예시 — StackManager 구현
public class StackManager : BaseCurrency
{
    protected override string SaveKey => "Stack";       // 코스트 이름
    protected override int startingAmount => 0;         // 시작값
}

// ✅ 올바른 예시 — HpManager 구현 (세부 사항 추가)
public class HpManager : BaseCurrency
{
    protected override string SaveKey => "PlayerHp";   // 코스트 이름
    protected override int startingAmount => 30;        // 시작값

    // 최대값 등 세부 사항은 여기서 추가
    public int MaxHp { get; private set; } = 30;

    public override void Add(int value)
    {
        base.Add(value);
        Amount = Mathf.Min(Amount, MaxHp); // 최대값 제한 추가
    }
}
```

### 주의사항
- `SaveKey`는 **고유해야** 한다 — 다른 클래스와 겹치면 PlayerPrefs 충돌 발생
- 기본 `Use()`·`Add()`로 충분하면 재정의 불필요, 필요할 때만 `override`

## 2) 열거형(enum) 작성 규칙

### 기본 원칙
`CardAffinity`, `CardType`처럼 게임 상태·분류를 나타내는 값은 `enum`으로 선언한다.

### 네이밍 규칙

```csharp
// ✅ 올바른 예시
public enum CardType       // PascalCase, 단수형
{
    None       = 0,        // 반드시 포함 (기본값 안전장치)
    Attacker   = 1,
    Defender   = 2,
    Supporter  = 3,
}

public enum CardAffinity
{
    None        = 0,
    Gambler     = 1,       // 의미 있는 단어, 약어 금지
    Safety      = 2,
    Opportunist = 3,
    Optimist    = 4,
}

// ❌ 잘못된 예시
public enum ctype { atk, def, sup }   // 소문자, 약어 사용
public enum CardTypes { ... }         // 복수형 사용
```

### 규칙 요약
- 이름은 `PascalCase`, **단수형**
- 멤버는 `PascalCase`, 의미를 알 수 있는 단어 사용 (약어 금지: `atk` ❌ → `Attacker` ✅)
- **`None = 0` 반드시 포함** — 기본값(0)이 의도치 않은 상태가 되는 것을 방지
- 멤버 값(`= 1, 2, 3...`) 명시 권장 — 나중에 중간 삽입/삭제 시 혼동 방지
- 관련 Helper(라벨, 범위, 색상 등)는 `static class <EnumName>Helper`로 분리 (예: `AffinityHelper`)

## 3) 변수·메서드 네이밍 규칙

### 변수 종류별 규칙

| 종류 | 규칙 | 예시 |
|---|---|---|
| `private` 필드 | `_camelCase` (언더스코어 접두사) | `_amount`, `_isDead` |
| `public` 프로퍼티 | `PascalCase` | `Amount`, `MaxHp` |
| 로컬 변수 | `camelCase` | `amount`, `targetEnemy` |
| `[SerializeField]` | `camelCase` | `amountText`, `enemyPrefab` |
| `const` 상수 | `PascalCase` 또는 `UPPER_SNAKE_CASE` | `MaxStack`, `MAX_STACK` |
| 이벤트 | `On + PascalCase` | `OnCurrencyChanged`, `OnDead` |
| 코루틴 | `동사 + Coroutine` 또는 `I + 동사` | `SpawnCoroutine`, `ISpawn` |

### 메서드 규칙
- 이름은 `PascalCase`, **동사로 시작**
- 반환값이 `bool`인 메서드는 `Is`, `Has`, `Can` 접두사 권장

```csharp
// ✅ 올바른 예시
public bool Use(int value)           // 동사 시작
public void Add(int value)
private void UpdateText(int amount)  // private도 PascalCase
public bool IsAlive()                // bool → Is 접두사
```

## 4) ScriptableObject(SO) 작성 규칙

- SO 클래스 파일은 `Assets/Scripts/<기능>/SO/` 폴더에 저장
- `[CreateAssetMenu]` 어트리뷰트 필수 (에디터에서 생성 가능하도록)
- 클래스명은 `Data` 또는 `SO` 접미사 권장 (예: `CardData`, `EnemySO`)

```csharp
// ✅ 올바른 예시
[CreateAssetMenu(menuName = "DarkHorror/CardData")]
public class CardData : ScriptableObject
{
    public CardType type;
    public CardAffinity affinity;
    public int cardPower;
}
```

## 5) 주석 규칙

- **클래스·public 멤버**는 `/// <summary>` XML 주석 권장
- **기획 연동 수치**는 반드시 기획서 원문 경로를 주석으로 명시
- 코드 블록 내 구분이 필요할 때 `// ---` 구분선 사용 (CardAffinity.cs 패턴 참고)

```csharp
/// <summary>
/// 동료 성향. 스택 기여량 생성 범위를 결정한다.
/// 기획 원문: 기획/시스템/03_동료_성향_세부기준.md
/// </summary>
public enum CardAffinity { ... }
```
