# 컬러 스타일 토큰 가이드

## 문서 목적
- Figma에서 바로 Color Style로 등록할 수 있는 기준 색상 체계를 정리한다.
- 화면마다 제각각 색을 쓰지 않도록 역할 중심으로 관리한다.

---

## 1) 기본 등록 방식
Figma Color Style 이름은 아래처럼 쓰는 걸 권장한다.

- `bg/default`
- `bg/elevated`
- `surface/panel`
- `surface/card`
- `line/default`
- `text/primary`
- `text/secondary`
- `accent/brass`
- `accent/teal`
- `accent/purple`
- `state/danger`
- `state/warning`
- `state/success`

---

## 2) 배경 계열
### `bg/default`
- `#0F1115`
- 전체 기본 배경

### `bg/soft`
- `#151821`
- 메인 배경 위 보조 구역

### `bg/elevated`
- `#1B1F27`
- 살짝 떠 있는 패널 뒤 배경

### `surface/panel`
- `#202631`
- 일반 패널 바탕

- `#262D38`

### `surface/steel`

- 금속성 포인트/슬롯 테두리

### `surface/leather`
- `#3F2E24`
- 가죽/문서/유물 파츠

## 6) 강조색 계열
### `accent/brass`
- `#C39A52`
- 황동, 핵심 포인트, 버튼 강조

### `accent/teal`
- `#4CB3B3`
- 유물 에너지, 시스템 활성 상태

### `accent/purple`
- `#7B5EA7`
- 봉인, 룬, 비술 계열 강조

---

## 7) 상태색 계열
### `state/danger`
- `#B3263E`
- 체력 저하, 치명 위기, 패널 경고

### `state/warning`
- `#C97A2B`
- 주의, 부족, 불안정

### `state/success`
- `#4F8F63`
- 회복, 적용 완료, 안정 상태

### `state/info`
- `#4C78A8`
- 정보성 안내

---

## 8) 사용 규칙
- 한 화면에서 강조색 주연은 1개, 보조는 1개만 쓴다.
- 경고/위험은 항상 `danger` 계열로 통일한다.
- 청록과 보라는 같이 써도 되지만 둘 다 강하게 쓰면 산만해진다.
- 버튼 기본색은 무채색/금속 계열, 활성 상태만 강조색 사용 권장.

---

## 9) 컴포넌트 적용 예시
### 패널
- Fill: `surface/panel`
- Stroke: `line/default`
- Header Accent: `line/brass`

### 기본 버튼
- Fill: `surface/panel-alt`
- Stroke: `line/strong`
- Text: `text/primary`

### 강조 버튼
- Fill: `accent/brass`
- Text: `bg/default`

### 위험 버튼
- Fill: `state/danger`
- Text: `text/primary`

### 카드
- Fill: `surface/card`
- Stroke: `line/default`
- Number/Text: `text/primary`
- Highlight: `accent/teal` 또는 `accent/purple`
