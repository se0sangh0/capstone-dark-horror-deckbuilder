# 피그마 디자인 톤앤매너 가이드

## 문서 목적
- 팀이 같은 분위기의 화면을 만들 수 있도록 비주얼 기준을 통일한다.
- 다크 호러 기반 위에 스팀펑크, SF, 판타지, 무협, 현대 오브젝트를 섞더라도 세계관이 분리되지 않게 한다.
- 피그마 시안 제작 시 바로 참고할 수 있는 실무 기준을 제공한다.

---

## 1) 전체 디자인 방향 한 줄
**"어두운 심연 속에 여러 시대와 장르의 유물이 뒤섞여 있는 금속성 탐험 UI"**

핵심 감정은 다음 4가지로 잡는다.
- 불길함
- 낡은 기계 장치의 묵직함
- 알 수 없는 유물의 신비감
- 여러 시대가 억지로 봉합된 듯한 이질감

즉, 예쁜 판타지 UI가 아니라:
- **어둡고 무겁고 낡았는데**
- 곳곳에 **정체 불명의 기술과 유물 흔적**이 박혀 있고
- 장르가 섞여 있어도 **하나의 위험한 세계**처럼 보여야 한다.

---

## 2) 장르 혼합 원칙
이 프로젝트는 장르를 많이 섞지만, 막 섞으면 산만해진다.
그래서 다음 규칙으로 통일한다.

### 기본 베이스
베이스는 항상 **다크 호러 + 산업 금속 UI**다.
- 어두운 배경
- 철, 황동, 검은 금속, 가죽, 유리
- 낡은 장치, 계기판, 인장, 주술 흔적

### 장르별 역할
#### A. 스팀펑크
- 톱니, 밸브, 게이지, 금속 프레임, 황동 장식
- UI 프레임 구조와 패널 테두리에 사용
- "기계 장치가 돌아간다"는 인상을 주는 장르 축

#### B. SF
- 홀로그램처럼 보이는 얇은 빛선
- 스캔 라인, 미약한 HUD 표시, 상태 표시선
- 너무 미래적 네온보다는 **고장난 유물 기술**처럼 사용

#### C. 판타지
- 문양, 인장, 룬, 봉인, 이세계 유물 느낌
- 카드, 스킬 아이콘, 보스 연출, 제단/교회 계열 노드에 사용

#### D. 무협
- 부적, 먹선, 인장, 도장, 고문서 질감, 경락/기 흐름 같은 선형 표현
- 캐릭터/직업 상징, 일부 스킬 효과선, 장식 문양에 제한적으로 사용
- 중국풍 그 자체보다 **동양적 봉인술/비술 흔적**에 가깝게 가져간다.

#### E. 현대
- 의료 장비, 경고 라벨, 산업 안전 스티커, 케이블, 전광 표시
- 상태 이상, 수치 정보, 장비 라벨, 위험 경고 UI에 사용
- 현대 장르가 너무 앞에 나오면 분위기가 깨지므로 보조 장르로만 사용

### 장르 혼합 규칙 요약
- 베이스는 다크 호러
- 구조는 스팀펑크
- 정보 표시는 SF
- 상징/신비는 판타지
- 이질적 장식은 무협
- 실용 디테일은 현대

즉, **한 화면 안에 모든 장르를 다 외치지 말고, 역할별로 나눠서 섞는다.**

## 3) 비주얼 키워드
피그마 작업 시 계속 떠올릴 키워드:
- 심연
- 봉인된 기계
- 검은 철판
- 황동 리벳
- 붉은 경고등
- 청록색 잔광
- 낡은 문양
- 혈흔보다는 말라붙은 오염
- 유리관 속 압력계
- 고장난 HUD
- 교단의 제단
- 이종 문명의 잔해

- 너무 채도 높은 네온은 피한다.

## 5) 재질 / 표면 표현 가이드
### 추천 재질
- 검은 철판
- 황동 테두리
- 긁힌 강철
- 오래된 가죽
- 금 간 유리
- 먼지 낀 양피지
- 마모된 페인트
- 케이블/배선/볼트

### 표면 표현 방식
- 완전 매끈한 플랫 UI보다는 **조금 낡은 표면감**이 있는 쪽을 우선한다.
- 다만 MVP에서는 텍스처를 과도하게 넣지 말고:
  - 단색 + 얇은 노이즈
  - 테두리 마모감
  - 살짝 어두운 그라데이션
정도로만 처리한다.

## 6) 레이아웃 톤
현재 문서 기준처럼 **고정 HUD + 중앙 패널 전환형** 구조를 유지한다.

### 레이아웃 인상
- 외곽 프레임은 금속 장치판처럼
- 중앙은 실제 플레이 구역처럼
- 좌측/상단 정보 패널은 계기판처럼
- 버튼은 기계 스위치 또는 봉인 해제 장치처럼 보이게

### 정보 우선순위
전투 화면에서는 아래 순서로 시선을 유도한다.
1. 현재 위험도 / 전황
2. 손패와 공용 스택
3. 동료 행동 순서
4. 적 상태
5. 세부 로그

즉, 예쁘기보다 **전투 판단이 먼저 보이는 구조**여야 한다.

## 7) 컴포넌트별 가이드
### A. 패널
- 모서리는 완전 둥글기보다 약간 각진 형태
- 금속 프레임 + 내부 어두운 판넬 구조
- 상단 라벨 바를 둬서 “전투 / 노드 / 용병소 / 교회”가 분명하게 보이게
- 얇은 빛선은 넣어도 되지만 과하면 SF UI처럼 튄다

### B. 버튼
- 기본 버튼: 금속 스위치/가압식 버튼 느낌
- 강조 버튼: 황동 + 붉은/청록 발광 포인트
- 위험 버튼: 어두운 적색 + 경고 표식
- 버튼 상태는 확실히 나뉘어야 함
  - 기본
  - hover
  - pressed
  - disabled

### C. 카드
- 한눈에 읽히는 숫자/속성 우선
- 장식보다 정보 전달이 먼저
- 카드 프레임은 역할/속성별로 색 포인트만 다르게
- 카드마다 장르를 과하게 다르게 주지 말고, 세계관 안의 다른 유물 카드처럼 통일감 유지

### D. 동료 슬롯
- 초상/아이콘 + 역할 + 상태를 바로 읽게 구성
- 전열/후열과 행동 순서가 직관적으로 보여야 함
- 체력/스트레스/역할 스택은 분리해서 읽히게

### E. 노드 아이콘
- 멀리서 봐도 구분되는 실루엣 우선
- 교회, 용병소, 전투, 이벤트 노드는 외곽 형태부터 달라야 함
- 세부 장르 요소는 아이콘 안에서만 녹이고, 기본 판별성은 유지

### F. 아이콘 스타일
- 채색 일러스트보다 단색/2색 기반 심볼형이 MVP에 적합
- 룬, 칼날, 톱니, 심장, 눈, 관, 제단, 전선, 탄피 같은 모티프 활용 가능
- 너무 귀엽거나 라운디한 아이콘은 피한다.

## 8) 타이포그래피 가이드
### 기본 방향
- 제목: 장식성 조금 있는 세리프 또는 고딕 계열
- 본문/수치: 읽기 쉬운 산세리프

### 인상
- 제목은 고풍/의식/기록물 느낌
- 본문과 수치는 현대 UI처럼 명확하게

### 규칙
- 수치와 정보 전달은 무조건 가독성 우선
- 제목 폰트가 과하면 UI 전체가 게임 포스터처럼 보여서 실사용성이 떨어진다.
- 한 화면에서 폰트 종류는 2종 이내 권장

## 9) 오브젝트 디자인 방향
상호가 원하는 “여러 장르 오브젝트 혼합”은 아래 방식이 좋다.

### 넣기 좋은 오브젝트 예시
- 스팀펑크: 압력계, 밸브, 리벳, 증기관, 기계식 손잡이
- SF: 고장난 스캐너, 에너지 코어, HUD 파편, 반투명 표시판
- 판타지: 봉인석, 룬판, 제단, 마도서, 유물 조각
- 무협: 부적, 인장, 도장, 먹선 문양, 경락도 같은 흐름선
- 현대: 카트리지, 주사기, 모니터 경고창, 택티컬 케이스, 산업 라벨

### 섞는 방식
- 한 오브젝트에 장르를 3개 이상 섞지 않는다.
- 주 오브젝트 1개 + 보조 디테일 1개 정도가 적당하다.
- 예시:
  - 황동 프레임 + 청록 에너지 코어
  - 낡은 제단 + 산업 경고 라벨
  - 가죽 문서철 + 룬 각인 + 금속 잠금장치

- 장르 요소를 한 화면에 전부 동등하게 때려 넣는 방식
- 텍스트보다 배경이 더 눈에 띄는 과장된 텍스처
- 의미 없는 장식선 남발

## 11) MVP용 실전 적용 원칙
첫 피그마 시안에서는 욕심내지 말고 아래만 먼저 잡는다.

### 1차 시안에서 꼭 필요한 것
- FHD(1920x1080) 기준 전투 화면 1장
- 고정 HUD와 중앙 전투 패널 구조
- 손패 / 스택 / 동료 배치 / 적 정보 / 로그 위치 확정
- 메인 컬러 3종 + 강조색 1~2종만 사용
- 버튼, 패널, 카드 1종씩 스타일 샘플 제작

### 1차 시안에서 굳이 안 해도 되는 것
- 모든 노드/아이콘 완성
- 세부 텍스처 과다 적용
- 화려한 이펙트 시뮬레이션
- 모든 직업군 비주얼 정의

## 12) 피그마 작업 순서 추천
1. 1920x1080 프레임 생성
2. 검은 금속 배경 + 패널 구획 잡기
3. 정보 구조 배치
4. 패널/버튼/카드 스타일 1차 정의
5. 강조색 적용
6. 장르별 오브젝트 2~3개만 샘플로 얹기
7. 팀 피드백 후 디테일 확장

## 13) 최종 체크 질문
시안 완성 후 아래 질문에 “예”가 나와야 한다.
- 이 화면은 한눈에 어둡고 불길한가?
- 하지만 게임 정보는 읽기 쉬운가?
- 장르가 섞였는데도 한 세계처럼 보이는가?
- 플레이어가 손패/스택/행동 순서를 바로 읽을 수 있는가?
- 장식보다 시스템 이해가 먼저 되는가?
