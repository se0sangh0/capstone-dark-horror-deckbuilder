# 제출용 작성 가이드 (PDF 양식 매핑)

아래는 제공된 PDF 양식에 **어디에 무엇을 넣을지** 바로 참고할 수 있도록 정리한 매핑표입니다.

---

## 1페이지: 표지

### 항목
- `제 목`
  - 입력: **괴이탐사국: 다크 호러 덱빌딩 오토배틀러 (가칭)**
- `팀명`
  - 입력: 팀 공식명
- `참여학생 (학번 / 성명 / 연락처)`
  - 입력: 팀원 전원 정보

### 작성 팁
- 제목은 너무 길면 `괴이탐사국 (가칭)`으로 줄이고, 본문에서 풀네임 표기
- 연락처는 학교 지침에 따라 개인 휴대폰 또는 이메일 중 하나로 통일

---

## 2페이지: 목차

PDF에 보이는 목차 구조에 맞춰 아래처럼 작성하면 됨.

1. 게임 개발 개요  
   1) 게임 제목  
   2) 플랫폼  
   3) 기획 의도  
   4) 장르  
   5) 차별화 요소  
   6) 게임 조작법  
   7) 클리어 조건  
   8) 주요 오브젝트

2. 게임 소개  
   1) 시놉시스 및 세계관  
   2) 플레이 컨셉  
   3) 그래픽 컨셉  
   4) 사운드 컨셉  
   5) UI/UX 컨셉

3. 게임 컨텐츠  
   1) 컨텐츠  
   2) 엔딩

4. 마무리  
   1) 기획 요약  
   2) 개발 게임 활용방안

---

## 3페이지: 게임 개발 과제 팀별 계획서

### 항목별 입력값
- `팀번호`: 수업에서 부여된 번호
- `팀명`: 표지와 동일
- `팀장학생명`: 팀장 이름
- `과제명`: 게임 개발 과제명(교과목 기준 명칭)
- `참여학생`: 학번/성명/연락처 표 채우기

### 과제계획서(핵심 3줄)
- `게임 개발 목표`
  - 예시: “덱빌딩-오토배틀러 하이브리드 핵심 루프를 MVP로 구현하고 플레이 테스트를 통해 재미 요소를 검증한다.”
- `핵심 기술`
  - 예시: “공용 스택 기반 전투 로직, 상태 관리(GameState), 패널 전환형 UI 아키텍처.”
- `게임 개발 내용`
  - 예시: “카드/동료/노드/스트레스 시스템 MVP 구현 후 향후 콘텐츠 확장 설계.”

---

## 4페이지: Ⅰ. 게임 개요

PDF 항목 순서에 맞춰 입력:

1) 게임 제목  
2) 플랫폼  
3) 기획 의도  
4) 시점  
5) 장르  
6) 재미 요소  
7) 게임 조작법  
8) 클리어 조건  
9) 주요 오브젝트

### 우리 프로젝트 기준 권장 문구
- `시점`: 2D UI 중심 탑뷰형 진행(전투/노드 패널 전환)
- `재미 요소`: 덱 설계 + 배치 순서 전략 + 스트레스 리스크 관리

---

## 붙여넣기용 최소 입력 템플릿

- 게임 제목: 괴이탐사국: 다크 호러 덱빌딩 오토배틀러 (가칭)
- 플랫폼: PC(Windows) 우선
- 기획 의도: 로그라이크 탐험과 덱빌딩·오토배틀러를 결합한 전략형 전투 경험 구현
- 시점: UI 중심 탑뷰형 진행
- 장르: 다크 호러 로그라이크 / 덱빌딩 / 오토배틀러
- 재미 요소: 공용 스택 설계, 동료 배치 순서 자동행동, 스트레스 시스템
- 조작법: 마우스 중심 + 단축키 보조
- 클리어 조건: MVP 테스트 시나리오 클리어(향후 정식 엔딩 확장)
- 주요 오브젝트: 카드, 동료(딜/탱/힐), 노드, 적, 재화(영혼석/마석), 스트레스
