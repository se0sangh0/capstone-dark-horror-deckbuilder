# 04_MVP_체크리스트_세부_v2

> 목적: MVP 구현/QA/제출까지 “누락 없이” 따라갈 수 있는 **세부형 체크리스트**.
> 
> 이 문서는 2026-03-22 기준 팀 합의(스택 3종/카드 무코스트/동료 2스킬/적 1~3명 순차 행동/PC 고정)를 반영한다.

---

## A. MVP 기준 고정(변경 금지)

### 우선순위 표기
- **(P0)**: 이번 주/이번 빌드에 반드시 들어가야 MVP가 성립
- **(P1)**: MVP에 포함되면 좋고, P0 완료 후 바로 진행
- **(P2)**: 선택/후순위(시간 남으면)


- [ ] **플랫폼**: PC 고정 **(P0)**
- [ ] **전투 핵심 컨셉**: 카드가 직접 공격하지 않고 **공용 스택 자원**을 생성한다. **(P0)**
- [ ] **스택 속성 3종(확정)**: 딜러 / 탱커 / 서포터 *(속성명은 추후 변경 가능)* **(P0)**
- [ ] **카드 코스트(확정)**: 없음 — 카드 = 스택 생성 전용 **(P0)**
- [ ] **동료 스킬(확정)**: MVP에서도 동료당 **스킬 2개** **(P0)**
  - [ ] MVP에서는 “모집 랜덤/중복 방지” 기능 **제외** **(P0)**
  - [ ] 출시 버전에서 모집 랜덤 + 중복 방지 적용(백로그로 유지) **(P0)**
- [ ] **턴 행동 규칙(확정)** **(P0)**
  - [ ] 모든 캐릭터는 **턴당 1회 행동** **(P0)**
  - [ ] 적이 3명이라면 **적 턴에 3명이 순서대로 1회씩 행동** **(P0)**
- [ ] **손패 규칙(확정)**: 매 턴 손패 리셋 + 사용 장수만큼 재드로우 **(P0)**
- [ ] **공용 스택 규칙(확정)** **(P0)**
  - [ ] 스택은 상한 없이 누적 **(P0)**
  - [ ] 동료가 행동 시 스택을 소비 **(P0)**
  - [ ] 스택 부족 시 해당 동료는 **대체 행동 없이 스킵** **(P0)**
- [ ] **미행동 보상(확정)** *(스킵 보상)* **(P0)**
  - [ ] 스킵된 동료: 역할 스택 +1 **(P0)**
  - [ ] 다음 턴 행동 순서 앞으로 당김 **(P0)**
    - 예: `1-2-3-4`에서 3이 스킵 → 다음 턴 `3-1-2-4`
- [ ] **배치 규칙(확정)**: 전투 전 배치 순서 확정 / 전투 중 변경 불가 **(P0)**
- [ ] **UI 기준 해상도(확정)**: FHD(1920x1080) **(P0)**

## B. 화면/UX 체크리스트(“무엇을 넣고 어떻게 띄울지”)

### B1. 타이틀/메인

- [ ] 타이틀 화면: 게임명/버전 표기 **(P1)**
- [ ] 메인 메뉴: 시작 / 설정 / 종료(또는 뒤로가기 처리) **(P1)**
- [ ] (선택) 조작법/튜토리얼 진입 버튼(1페이지 팝업) **(P2)**

### B2. 맵/노드(런 진행)

- [ ] 노드 4종 UI: 전투 / 휴식 / 상점 / 이벤트 **(P1)**
- [ ] 노드 선택 → 해당 씬/화면으로 전환 **(P1)**
- [ ] 현재 스트레스(또는 상태) 요약 표시(작게) **(P1)**

### B3. 전투 UI(핵심)

- [ ] **손패 패널** **(P0)**
  - [ ] 카드 3종 표시(아이콘 + 속성/수치) **(P0)**
  - [ ] 카드 사용 피드백(사용됨/비활성/선택 강조) **(P0)**
- [ ] **공용 스택 패널** **(P0)**
  - [ ] 딜러/탱커/서포터 3종 수치가 항상 보이게 **(P0)**
  - [ ] 카드 사용 시 즉시 증가 표시(숫자 애니메이션/색 강조 등) **(P0)**
- [ ] **아군(동료) 패널** **(P0)**
  - [ ] 배치 순서 표시(1~N) **(P0)**
  - [ ] 현재 턴 행동 예정 표시(“다음 행동자” 강조) **(P0)**
  - [ ] 동료별 스킬 2개 중 “어떤 스킬을 썼는지” 표시(텍스트 1줄이면 OK) **(P0)**
- [ ] **적 패널** **(P0)**
  - [ ] 적 1~3명 표시 + HP(최소) **(P0)**
  - [ ] 적 행동 순서 표시(적이 3명일 때 순차 행동이 보이게) **(P0)**
- [ ] **행동 로그(필수)** **(P0)**
  - [ ] “누가 어떤 스택을 얼마나 소비했고 무슨 효과가 났는지” 1~3줄 로그 **(P0)**
  - [ ] 스킵 발생 시 로그: “스택 부족으로 스킵” 명시 **(P0)**
- [ ] **턴/페이즈 표시** **(P0)**
  - [ ] TURN n **(P0)**
  - [ ] Phase: 카드 → 아군 행동 → 적 행동 → 결과 **(P0)**
- [ ] **스트레스 UI** **(P0)**
  - [ ] 증가/감소가 즉시 피드백 되게 **(P0)**
  - [ ] 디버프 발동 시 아이콘/문구 표시 **(P0)**

### B4. 전투 결과 UI

- [ ] 승리/패배 분기 **(P1)**
- [ ] 보상/패널티 표시(간단) **(P1)**
- [ ] “다음으로” 버튼 → 맵 복귀 **(P1)**

## C. 전투 시스템 구현 체크리스트(로직 단위)

### C1. 전투 상태머신/페이즈

- [ ] 전투 시작: 아군/적 스폰 + 스택 초기화(0) **(P0)**
- [ ] 턴 시작 **(P0)**
  - [ ] 손패 리셋 **(P0)**
  - [ ] 사용 장수 기록 초기화 **(P0)**
- [ ] 카드 페이즈 **(P0)**
  - [ ] 카드 사용 → 스택 증가 **(P0)**
  - [ ] 0장 사용도 가능 **(P0)**
- [ ] 아군 행동 페이즈 **(P0)**
  - [ ] 배치 순서대로 각 동료 1회 행동 **(P0)**
  - [ ] 스킬 2개 중 선택 규칙(우선순위) 적용 **(P0)**
  - [ ] 스택 부족 → 스킵 처리 **(P0)**
- [ ] 적 행동 페이즈 **(P0)**
  - [ ] 적이 1~3명일 때 “존재하는 적”이 순서대로 1회 행동 **(P0)**
- [ ] 결과 처리 **(P0)**
  - [ ] 사망/승패 판정 **(P0)**
  - [ ] 스트레스 변화 적용 **(P0)**
- [ ] 턴 종료 **(P0)**
  - [ ] 이번 턴 카드 사용 장수만큼 다음 턴 재드로우에 반영 **(P0)**
- [ ] 다음 턴 진입 또는 전투 종료 **(P0)**

### C2. 손패/드로우 규칙

- [ ] “턴 시작 시 손패 리셋” 구현 **(P0)**
- [ ] “사용 장수만큼 재드로우” 구현 **(P0)**
- [ ] 덱 소진 처리(최소) **(P0)**
  - [ ] 더 이상 드로우할 카드가 없을 때 규칙 정의/구현(탈진 패널티 등) **(P0)**

### C3. 공용 스택(3속성)

- [ ] 스택 데이터 구조 확정: 딜러/탱커/서포터 3개 값 **(P0)**
- [ ] 누적 규칙: 카드로 증가 **(P0)**
- [ ] 소비 규칙: 동료 스킬이 소비 **(P0)**
- [ ] 부족 규칙: 부족하면 “소비 안 하고 스킵” **(P0)**

### C4. 동료 스킬 2개(고정)

- [ ] 동료 스킬 A: 필요 스택(종류/수치) + 효과 정의 **(P0)**
- [ ] 동료 스킬 B: 필요 스택(종류/수치) + 효과 정의 **(P0)**
- [ ] 스킬 선택 우선순위 규칙(단순 룰 1줄로 고정) **(P0)**
  - 예: “가능한 스킬 중 더 큰 소비 우선” 또는 “A 우선, 불가하면 B”
- [ ] 스킬 실행 결과가 UI/로그에 남음 **(P0)**

### C5. 미행동 보상(스킵 처리 후속)

- [ ] 스킵 판정 조건: 스택 부족 **(P0)**
- [ ] 스킵된 동료에 역할 스택 +1 적용 **(P0)**
- [ ] 다음 턴 행동 순서 재정렬 적용 **(P0)**
- [ ] UI에 “순서 변경”이 눈에 보이게 표시 **(P0)**

## D. 콘텐츠(MVP 수량 고정) 체크리스트

### D1. 카드 3종

- [ ] 카드 1: 생성 스택(딜/탱/서포 중 1~2종) + 수치 확정 **(P1)**
- [ ] 카드 2: 동일 **(P1)**
- [ ] 카드 3: 동일 **(P1)**
- [ ] 카드 텍스트/아이콘/색상(속성 구분) 적용 **(P1)**

### D2. 동료 6종(각 2스킬)

- [ ] 동료 1~6: 역할 지정(딜/탱/서포) **(P1)**
- [ ] 동료 1~6: 스탯(HP 등 최소) 확정 **(P1)**
- [ ] 동료 1~6: 스킬 2개 정의(소비 조건/효과) **(P1)**
- [ ] 배치 순서 확정 방식(MVP) **(P1)**
  - [ ] 옵션 A: 전투 전 배치 UI 제공 **(P1)**
  - [ ] 옵션 B: MVP는 고정 순서로 시작(문서에 순서 명시) **(P1)**

### D3. 적 6종(3/2/1)

- [ ] 약한 적 3: 턴당 1회 행동 패턴 **(P1)**
- [ ] 중간 적 2: 패턴 2개 중 조건부 사용(간단) **(P1)**
- [ ] 보스 1: 조건부 변화(HP 특정 이하 시 패턴 변경 등) 1개 이상 **(P1)**

---

## E. 노드 4종(런 연결)

- [ ] 전투 노드: 전투로 진입 **(P1)**
- [ ] 휴식 노드: 스트레스 감소(고정 수치로 MVP 단순화) **(P1)**
- [ ] 상점 노드: (MVP) 회복 or 카드 교체 중 1개만이라도 구현 **(P1)**
- [ ] 이벤트 노드: 선택지 2개(보상/패널티)로 최소 구현 **(P1)**

## F. QA/완성 기준(“돌아가는 플레이 가능 프로토타입”)

- [ ] “메인 → 맵 → 전투 → 결과 → 맵” 루프가 끊기지 않는다 **(P0)**
- [ ] 적 3명 전투에서 “적 턴 3명 순차 행동”이 정확히 보이고 동작한다 **(P0)**
- [ ] 스택 부족 시 스킵 + 미행동 보상이 정상 작동한다(로그/표시 포함) **(P0)**
- [ ] 보스 1회 포함 플레이스루 완주 가능 **(P0)**
- [ ] 크래시/소프트락/입력 막힘 없음 **(P0)**
- [ ] 시연 동선(1~2분) 문서화 **(P0)**
