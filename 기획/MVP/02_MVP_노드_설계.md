# MVP 노드 설계

## 노드 구조

- 총 10층, 각 층마다 **3개 노드 분기 중 1개 선택**
- 총 방문 노드: **10개** (층당 1개 × 10층)
- 1층, 9층(화툿불), 10층(보스) 고정

---

| 구분 | 노드 | 우선순위 | 내용 공개 여부 |
|---|---|---|---|
| 오픈 노드 | 전투 | ✅ 우선 구현 | 진입 전 공개 |
| 오픈 노드 | 화툿불 | ✅ 우선 구현 | 진입 전 공개 |
| 오픈 노드 | 보스 | ✅ 우선 구현 | 진입 전 공개 |
| 이벤트 노드 | 용병소 | ✅ 우선 구현 | 진입 전 비공개 |
| 이벤트 노드 | 교회 | ⏳ 백로그 | 진입 전 비공개 |
| 이벤트 노드 | 엘리트 전투 | ⏳ 백로그 | 진입 전 비공개 |

---

## 오픈 노드 상세

### 전투 노드
- **역할**: 카드 → 공용 스택 → 동료 자동행동의 기본 루프 증명
- **MVP 필수 이유**: 게임의 핵심 차별점이 가장 직접적으로 드러난다.
- **구현 메모**: 일반 적 3종과 먼저 연결

### 화툿불 노드
- **역할**: 체력/스트레스 회복, 다음 전투 전 정비
- **MVP 필수 이유**: 전투 사이 숨 고르기 구간. 스트레스 시스템 검증에 필요.
- **구현 메모**: 무료 회복량 수치는 밸런스 테스트 후 확정 (-15 임시값)

### 보스 노드
- **역할**: 런 종료 조건, 난이도 기준점, 발표용 데모 목표
- **MVP 필수 이유**: 15~20분 완주 기준을 세우려면 명확한 종착점이 필요하다.
- **구현 메모**: 보스 1종과 고정 진입 연출만 먼저 구현

---

## 이벤트 노드 상세

### 용병소
- **역할**: 영혼석 소비, 동료 모집, 파티 구성 변화
- **MVP 필수 이유**: 동료 수 증가에 따른 손패 상한 증가와 덱 확장 규칙을 보여줄 수 있다.
- **구현 메모**: 초반에는 고정 후보 2~3명 중 선택 구조로 단순화

### 교회 ⏳ 백로그
- **역할**: 체력 회복, 스트레스 관리, 사망 동료 부활
- **구현 메모**: 무료 스트레스 관리(-25 임시값)와 유료 부활/회복 분리
- **후순위 이유**: 화툿불이 회복/안정 기능을 먼저 대체 가능

### 엘리트 전투 ⏳ 백로그
- **역할**: 일반 전투보다 강한 적, 추가 보상(영혼석/아이템)
- **구현 메모**: 엘리트 적 2종 연결
- **후순위 이유**: 기본 전투 루프 검증 후 난이도 분기 추가

---

## MVP 추천 노드 시퀀스
- 예시 루프 A: 전투 → 용병소 → 전투 → 화툿불 → 전투 → 보스
- 예시 루프 B: 전투 → 화툿불 → 전투 → 용병소 → 전투 → 보스
