# MVP 노드 4종 우선안

## 추천 결론
MVP 우선 구현 노드는 아래 4종으로 잡는 것을 권장한다.

1. 전투 노드
2. 용병소
3. 교회
4. 보스 노드

## 선정 기준
- 전투 루프를 반드시 증명할 수 있어야 한다.
- 동료 모집과 재화 소비가 보여야 한다.
- 스트레스/회복/부활 시스템이 최소 1개 노드에서 작동해야 한다.
- 최종 목표 노드가 있어야 플레이 루프 1회 완주 기준을 만들 수 있다.

## 노드별 역할

### 1. 전투 노드
- 역할
  - 카드 -> 공용 스택 -> 동료 자동행동의 기본 루프 증명
- MVP 필수 이유
  - 게임의 핵심 차별점이 가장 직접적으로 드러난다.
- 구현 메모
  - 일반 적 3종과 먼저 연결

### 2. 용병소
- 역할
  - 영혼석 소비, 동료 모집, 파티 구성 변화
- MVP 필수 이유
  - 동료 수 증가에 따른 손패 상한 증가와 덱 확장 규칙을 보여줄 수 있다.
- 구현 메모
  - 초반에는 고정 후보 2~3명 중 선택 구조로 단순화

### 3. 교회
- 역할
  - 체력 회복, 스트레스 관리, 사망 동료 부활
- MVP 필수 이유
  - 다크 호러 감정 자원인 스트레스가 실제로 플레이 결정을 흔드는지 검증할 수 있다.
- 구현 메모
  - 무료 스트레스 관리와 유료 부활/회복을 분리

### 4. 보스 노드
- 역할
  - 런 종료 조건, 난이도 기준점, 발표용 데모 목표
- MVP 필수 이유
  - 15~20분 완주 기준을 세우려면 명확한 종착점이 필요하다.
- 구현 메모
  - 보스 1종과 고정 진입 연출만 먼저 만든다.

## 후순위 노드

### 화톳불 노드
- 장점
  - 보스 전 정비 흐름이 직관적
- 후순위 이유
  - 교회가 회복/안정 기능을 먼저 대체 가능

### 약탈 노드
- 장점
  - 리스크/보상 선택이 살아난다.
- 후순위 이유
  - MVP에서는 분기 복잡도만 늘릴 가능성이 큼

### 기타 히든 노드
- 후순위 이유
  - 컨셉 표현에는 좋지만 기본 루프 증명보다 우선순위가 낮다.

## MVP 추천 노드 시퀀스
- 예시 루프 A
  - 전투 -> 용병소 -> 전투 -> 교회 -> 전투 -> 보스
- 예시 루프 B
  - 전투 -> 교회 -> 전투 -> 용병소 -> 전투 -> 보스

## 구현 순서
1. 전투 노드
2. 교회
3. 용병소
4. 보스 노드

## 결정 필요 항목
- 전투 노드와 보스 노드 사이 최소 거리
- 교회의 무료 스트레스 회복량
- 용병소의 모집 비용과 후보 수
- 보스 진입 전 강제 회복 이벤트를 둘지 여부
