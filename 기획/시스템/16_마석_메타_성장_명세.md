## 1. 개요

**마석(Magic Stone)** 은 런(run)을 넘어 유지되는 **영구 메타 재화**다. 마석으로 **영구 패시브/스킬**을 해금하며, 해금 효과는 이후 모든 런에 상시 적용된다.

원래 계획한 **마을 시스템은 시간상 보류**하고, 그 역할을 **로그라이크 메타 루프**로 대체한다:

> 보스 클리어(또는 전멸) → 모든 런 데이터 리셋 → 마석 누적 유지 → **패시브 해금 화면** → 새 런 첫 배틀 노드부터 재시작.

게임은 메인 메뉴로 돌아가지 않고 **연속 루프**로 굴러가며, 마석/해금만 영속해 회차를 거듭할수록 강해진다.

## 2. 런 루프 (2026-06-02 핵심 변경)

### 2-1. 기존
- 보스 클리어 → 엔딩 → `GameStartScene`(메인 메뉴) 복귀.
- 전멸 → `GameStartScene` 복귀.

### 2-2. 변경 (로그라이크 루프)
보스 클리어 **및 전멸 모두 동일 루프**:

1. **리셋**: 예비대(대기석) 초기화 + 활성 파티 초기화(기본 파티) + 영혼석 기본값(20) 리셋.
2. **마석 유지**: 마석은 리셋하지 않고 누적 유지. (별도 클리어 보너스 없음 — 마석은 런 중 노드 보상으로만 적립: Combat +10 / Elite +20 / Boss +30, 기존 사양 유지.)
3. **패시브 해금 화면**: 첫 배틀 노드 진입 **전**에 마석 상점(패시브 해금) 표시. (초록 시작 노드와 동일 패널 재사용.)
4. **새 런 시작**: 노드맵 재생성 → 첫 배틀 노드부터.

> 메인 메뉴(`GameStartScene`)는 최초 실행/튜토리얼 진입 용도로만 유지.

## 3. 마석 해금 — 카테고리 A: 동료 시그니처 패시브 (전투 관여형, 풀+랜덤 배정)

스탯/재화 증가가 아니라 **전투 메커니즘에 관여하는 상시 능력**(롤 패시브 / 포켓몬 특성형). 각 동료 **컨셉 + 밸런스** 기반.

### 3-0. 보유/배정 모델 (2026-06-02 확정)
- 각 동료는 **3개의 패시브 풀**을 가진다(총 5동료 × 3 = 15종).
- 패시브는 개별 해금(마석). 해금하면 그 동료의 **"해금된 풀"** 에 들어간다.
- **배정**: 동료 인스턴스가 생성되는 **런 시작 시 1회**, 해당 동료의 해금된 풀에서 **무작위 1개**를 `activePassiveId` 로 배정(전투마다 재배정 안 함). 해금된 게 없으면 패시브 없음.
- **표시**: 인스턴스마다 다르므로 **카드에 현재 활성 패시브 표시**(CardSlotView/BattleCardView).
- **전투 적용**: 각 훅에서 `user.activePassiveId == <id>` 로 발동 판정(전역 IsUnlocked 아님).

### 3-1. 패시브 목록 (동료당 3개)

## 4. 마석 해금 — 카테고리 B: 동료 시그니처 스킬 해금

각 동료의 **시그니처(고코스트) 스킬**을 마석으로 해금. **미해금 시** 해당 동료의 스킬 풀(4개)에서 제외되어 배정되지 않고, 해금 시 풀에 추가된다(기존 "4개 풀 중 2개 랜덤" 체계 유지 — 해금 스킬은 선택 후보에 포함).

| ID | 동료 | 해금 스킬 | 비용 |
|---|---|---|---|
| `unlock_skill_ice_storm`  | 캐스터 | 아이스스톰 (6코, 광역 55) | 60 |
| `unlock_skill_sky_slash`  | 오펜더 | 하늘가르기 (10코, 단일 75) | 60 |
| `unlock_skill_war_shield` | 디펜더 | 전장의 방패 (6코, 데미지+전체실드) | 60 |
| `unlock_skill_war_cry`    | 어택커 | 워크라이 (5코, 데미지+도발) | 60 |
| `unlock_skill_prayer`     | 프리스트 | 기원 (5코, 전체힐 40) | 60 |

- 적용 지점: 동료 스킬 배정부(`BattleManager.cs` `skillPool` 2-of-4 추출) 직전, `MetaPassiveManager.IsSkillUnlocked(skillId)` 가 false 인 시그니처 스킬을 풀에서 제외.
- 시그니처 스킬이 아닌 일반 스킬은 항상 풀에 포함(해금 불필요).

## 5. 마석 경제 / 비용 요약

- 획득: 노드 보상(기존) — Combat +10 / Elite +20 / Boss +30. 별도 클리어 보너스 없음.
- 총 해금 비용: 패시브 5종(80+80+100+90+80=430) + 스킬 5종(60×5=300) = **730 마석**.
- 한 런 ~80~120 마석 가정 → 전체 해금까지 대략 6~9런. 회차형 장기 성장 곡선.

## 6. 저장 / 영속성

| 데이터 | 키 | 형식 |
|---|---|---|
| 마석 잔액 | `ManaStone`(기존) | int |
| 각 해금(패시브/스킬) | 위 표의 ID | int 0/1 |

- 해금 상태는 PlayerPrefs 영속. 디버그 초기화 1수단(설정 팝업 재화 초기화 확장 — `MetaPassiveManager.ResetAll`).

## 7. UI — 마석 상점 (패시브 해금)

- 진입: **노드맵 초록 시작 노드**(상시) + **런 루프의 첫 배틀 전 화면**(자동 표시). 동일 `MagicStoneShopPanel` 재사용.
- 구성: 보유 마석 + 카드 10종(패시브 5 + 스킬 5) — 항목이 많으므로 **스크롤 또는 2열 그리드**. 각 카드: 이름/효과/비용 + 상태 버튼(해금/마석부족 비활성/해금완료).
- 닫기 → 노드맵 복귀(상시 진입) 또는 첫 노드 진행(루프 진입).

---

## 8. 폐기 / 대체

- (폐기) 초안의 보상형 패시브 3종 — `meta_soul_gain`(영혼석+25%) / `meta_post_heal`(전투후 HP회복) / `meta_calm`(스트레스 회복 2배). 스탯/재화 증가 방향을 버리고 전투 관여형 + 스킬 해금으로 전면 대체.
- (폐기) 마석→영혼석 10:1 변환(이미 제거됨, §이전 버전).
