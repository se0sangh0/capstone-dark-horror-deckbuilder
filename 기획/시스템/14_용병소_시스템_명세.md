## 1. 개요

용병소는 탐색 중 등장하는 이벤트 노드로, 파티가 전투를 이어가기 위해 필요한 동료를 모집하고 교체하는 장소다.

이 시스템의 핵심은 다음 3가지다.
- 전투 중 사라진 빈 슬롯을 복구한다.
- 현재 파티의 조합을 조정한다.
- 새로운 동료를 영입해 전투 전력을 보강한다.

MVP에서는 복잡한 상점형 성장 시스템이 아니라, "동료를 고르고 / 비용을 지불하고 / 파티를 정리하는" 단순한 회복·보강 허브로 설계한다.

---

## 2. 기획 의도

### WHY
- 전투에서 동료가 사망하면 파티 전력이 즉시 감소하므로, 이를 복구할 수 있는 중간 허브가 필요하다.
- 단순 전투 반복만 있으면 런의 변동성이 너무 낙아진다.
- 파티 조합을 바꾸는 선택지를 줘서, 망한 판도 어느 정도 되살릴 수 있어야 한다.

### WHO
- 탐색 노드에서 용병소 이벤트를 만난 플레이어
- 전투 후 전력 손실을 복구하려는 플레이어
- 파티 구성을 조정하려는 플레이어

### WHAT
- 후보 동료 목록을 보여준다.
- 영혼석을 지불해 동료를 고용한다.
- 후보 목록을 다시 뉵는다.
- 현재 파티를 재정비한다.

### SUCCESS
- 빈 파티 슬롯을 자연스럽게 메울 수 있다.
- 플레이어가 "누굴 뉵아야 하는지"를 명확히 판단할 수 있다.
- 비용과 보상이 너무 무겁지 않아, 반드시 들러야 하지만 항상 이득만 보는 노드는 아니어야 한다.

## 5. 기본 규칙

### 5-1. 노드 진입 규칙
- 용병소는 맵의 이벤트 노드로 등장한다.
- 노드 클릭 시 `currentView = 'mercenary'` 로 전환한다.
- 진입 후에는 용병소 전용 UI가 열린다.

### 5-2. 후보 표시 규칙
- 용병소 진입 시 기본적으로 후보 동료 3인을 표시한다.
- 후보는 `reserves` 또는 별도 후보 풀에서 생성한다.
- 각 후보는 이름, 직업, 역할, 모집 비용, 성향을 표시한다.
- 후보 목록은 한 화면에서 비교 가능해야 한다.
- 추후 패시브 시스템으로 대기석 슬롯이 증가할 수 있다.

### 5-3. 고용 규칙
- 플레이어는 대기석에 있는 동료를 선택해 고용할 수 있다.
- 고용 시 소울스톤이 모집 비용만큼 차감된다.
- 1성 동료의 모집 비용은 30이다.
- 동료 성급이 올라가면 성급당 비용이 추가로 증가한다.
- 승급에 따른 추가 비용 증가 규칙은 아직 미정이며, 확장 항목으로 남긴다.
	- 현재는 추가 비용 증가 없이 스킭 관련 메리트만 존재하는 상태로 구현한다.
- 고용 성공 시 파티 또는 후보 대기열에 반영한다.
- 파티가 4명 미만이면 빈 슬롯을 우선 채울 수 있다.
- 파티가 4명일 경우, 교체 모드로 전환한다.
- 후보 대기열의 수는 9칸이며 용병소 노드에서 나가면 리셋된다.

### 5-4. 리롤 규칙
- 플레이어는 일정 비용을 지불해 후보 목록을 다시 뉵을 수 있다.
- 리롤 후 기존 후보는 폐기되고 새 후보 3인으로 교체된다.
- 리롤 비용은 해당 용병소 노드 내에서만 누적된다.
- 최초 리롤 비용은 영혼석 2다.
- 같은 노드에서 두 번째 리롤부터는 직전 비용에 +1씩 증가한다. (2 → 3 → 4 ...)
- 노드를 벗어나면 리롤 비용 누적 상태는 초기화된다.
- 리롤은 무한 반복 가능하게 두되, 밸런스 조정 시 상한을 둘 수 있다.

### 5-5. 파티 편집 규칙
- 용병소에서는 파티 편집 UI로 진입할 수 있다.
- 파티 편집은 전투 중이 아닐 때만 가능하다.
- 파티 구성 화면에서는 현재 파티 4칸과 대기열 동료 목록을 함께 보여준다.
- 파티원은 위치 순서를 변경할 수 있다.
- 대기열 동료를 선택해서 교체할 수 있다.
- 빈 슬롯이 있으면 대기열 동료를 바로 배치할 수 있다.
- 파티원이 가득 찼 경우에는 기존 파티원 1명과 대기열 동료 1명을 교체하는 방식으로 처리한다.
- 편집 후 현재 파티 정보는 즉시 반영된다.
- 파티 편집 페이지에서는 현재 덱에 성향 및 역할 구성을 보여준다.
- (추후 확장) 직업 시너지 및 성향 시너지 정보를 볼 수 있는 대시보드가 존재한다.

## 6. 상세 동작 흐름

### 6-1. 진입 흐름

```text
ExplorePanel
  └─ event 노드 클릭
      └─ onNavigate('mercenary')
          └─ currentView = 'mercenary'
              └─ MercenaryOfficePanel 렌더
```

### 6-2. 고용 흐름

```text
후보 선택
  └─ 모집 버튼 클릭
      └─ 비용 확인
          ├─ 부족함 → 실패 메시지
          └─ 충분함 → 소울스톤 차감
              └─ 동료 획득
                  ├─ 빈 슬롯 있음 → 즉시 합류
                  └─ 파티 만석 → 교체 대상 선택
```

### 6-3. 리롤 흐름

```text
리롤 버튼 클릭
  └─ 리롤 비용 확인
      ├─ 부족함 → 실패 메시지
      └─ 충분함 → 소울스톤 차감
          └─ 후보 3인 재생성
```

### 6-4. 교체 흐름

```text
파티 만석 상태에서 고용
  └─ 교체 대상 선택
      └─ 기존 파티원 제거
          └─ 신규 동료 합류
              └─ 파티 목록 갱신
```

## 7. 예외 처리

### 7-1. 영혼석 부족
- 고용 또는 리롤 비용이 부족하면 실행하지 않는다.
- 실패 사유는 명확한 메시지로 표시한다.

### 7-2. 파티 만석
- 파티가 4명일 때는 자동 합류하지 않는다.
- 반드시 기존 파티원 1명을 교체 대상으로 선택해야 한다.

### 7-3. 빈 슬롯 처리
- 전투 중 동료 사망으로 생긴 빈 슬롯은 용병소에서만 복구한다.
- 빈 슬롯이 있으면 교체가 아니라 즉시 채우기 흐름을 우선한다.

### 7-4. 중복 직업 처리
- 중복 직업 편성은 허용한다.
- 같은 역할군이 여러 명이어도 고용 가능하다.

### 7-5. 후보 수 부족
- 후보 생성 실패 또는 데이터 부족 시, 기본 후보군을 사용한다.
- MVP에서는 항상 3명의 후보가 보이도록 보정한다.

## 8. UI 구성

### 8-1. 메인 패널
- 상단: 현재 소울스톤 / 파티 상태
- 상단 또는 좌측: 2개 메뉴 탭
  - 파티 모집
  - 파티 구성
  - ~~동료 성장~~ **(MVP 배제 — 백로그)**
- 중앙: 현재 선택한 메뉴의 내용
- 우측 또는 하단: 고용 / 리롤 / 닫기 버튼
- 부가 영역: 선택한 후보 상세 정보

### 8-2. 주요 인터랙션
- 메뉴 클릭: 파티 모집 / 파티 구성 화면 전환
- 후보 클릭: 상세 정보 표시
- 고용 버튼: 선택 후보 모집
- 리롤 버튼: 후보 재생성
- 파티 편집 버튼: 현재 파티 조정 모드 진입
- 닫기 버튼: 탐색 화면 복귀

### 8-3. 피드백 규칙
- 비용 부족: 빨간색 경고
- 고용 성공: 짧은 성공 알림
- 리롤 성공: 후보 목록 즉시 갱신
- 교체 성공: 현재 파티 목록 즉시 반영

## 10. 밸런스 기준

### 10-1. 역할
- 용병소는 보상 노드가 아니라 "복구 노드"다.
- 플레이어가 손해를 일부 회복할 수 있지만, 무조건 이득은 아니다.

### 10-2. 비용 원칙
- 빈 슬롯 복구는 부담 없이 가능해야 한다.
- 추가 고용은 일정 비용을 요구해야 한다.
- 리롤은 선택적 소비가 되어야 한다.
- 리롤 비용은 노드 내 반복을 억제할 정도로만 증가한다.

### 10-3. 추천 방향
- 후보 3인 중 최소 1명은 현재 파티에 의미 있는 선택지가 되도록 구성한다.
- 너무 강한 후보가 연속으로 나오지 않도록 한다.
- 빈 슬롯이 하나도 없더라도, 파티 압축/정리 선택이 생기면 런 전략성이 살아난다.
