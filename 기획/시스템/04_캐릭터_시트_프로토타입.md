## 2. 범위 / 단일 스킬 규칙

- `SingleEnemy` → power 그대로 적 1명에게 적용
- `AllEnemies` → power 그대로 적 전체에게 각각 적용
- 범위 스킬은 단일 스킬보다 코스트가 높은 것으로 차별화

## 5. SO 구조 확정

### MVP 구조 (현재)

```
AllyDefinition (ScriptableObject)
├── string id                      // ally_caster_01
├── string displayName             // 런타임에서 이름 생성기가 세팅
├── JobClass jobClass              // Caster / Offenser / Defender ...
├── Role role                      // Dealer / Tank / Support
├── int sequence                   // 직업군 내 순번
├── int maxHp
├── int stressResist
├── int recruitCost
├── SkillDefinition skill1         // SO 레퍼런스 (MVP 고정)
├── SkillDefinition skill2         // SO 레퍼런스 (MVP 고정)
├── AllyNameTableSO maleNameTable
├── AllyNameTableSO femaleNameTable
├── Gender gender                  // Male / Female / Random
└── Sprite portrait                // 선택
```

```
SkillDefinition (ScriptableObject)
├── string id
├── string displayName
├── string skillGroup              // "단일" / "범위"
├── StackType costType             // Dealer / Tank / Support
├── int costAmount
├── Targeting targeting            // SingleEnemy / AllEnemies / SingleAlly / AllAllies / Self
├── EffectType effectType          // Damage / Heal / Shield / Buff / Debuff
├── int power
├── int aiPriority
├── StatusEffect statusEffect      // None / Bleed / Taunt / StressHeal (선택 — 확장용)
├── int statusValue                // 선택
└── string description             // 선택
```

```
AllyNameTableSO (ScriptableObject)
├── string[] firstNames            // 10개
└── string[] lastNames             // 10개
```

### 실제 구현 전환 시 (스킬 다차원 리스트)

```
SkillPoolSO (ScriptableObject)
├── JobClass jobClass
├── SkillDefinition[] commonSkills
└── SkillDefinition[] rareSkills   // 확장용

AllyDefinition (변경)
├── SkillPoolSO skillPool          // skill1, skill2 → 풀로 교체
└── int skillSlotCount = 2
```

> MVP에서 skill1/skill2 직접 참조 → 실제 구현 시 SkillPoolSO 방식으로 교체.
> SO 구조 자체를 재설계하지 않아도 됨.

## 6. 이름 생성 규칙

- ID: `ally_{jobClass}_{sequence}` (예: `ally_caster_01`, `ally_caster_02`)
- 시퀀스는 직업군별 독립 카운팅
- displayName: 런타임에서 `AllyNameGenerator`가 gender 기준으로 테이블 선택 후 조합
