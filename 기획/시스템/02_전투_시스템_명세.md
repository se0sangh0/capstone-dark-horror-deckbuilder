# 전투 시스템 명세

## 핵심 리소스
- 공용 스택: 플레이어 카드가 만들어내는 전투 자원
- 생성: 플레이어 카드
- 소비: 동료 행동
- 상한: 없음
- 잉여 처리: 전투 중 자동으로 소모되는 흐름을 기본 전제로 설계
- 부족 처리: 스택이 부족하면 해당 동료 행동은 무조건 스킵

## 턴 구조
1) 턴 시작 및 손패 리셋
2) 플레이어 카드 단계
3) 선공 판정 단계 (매 턴)
4) 선공 측 행동 단계
5) 후공 측 행동 단계
6) 스트레스/상태이상/사망 처리
7) 종료/상태 갱신

> (MVP 체크) 각 동료/적은 **턴당 1회 행동**을 기본으로 한다. (적이 3명이면 적은 순차로 3회 행동)

## 카드 단계 규칙 (2026-03-09 확정)
- 손패는 매 턴 리셋한다.
- 사용한 카드 수만큼 다시 드로우한다.
- 턴당 카드 사용 장수 제한은 두지 않는다.
- 더 이상 드로우할 카드가 없으면 탈진 번데미지를 받는다.

## 동료 행동 규칙 (2026-03-09 확정)
- 동료는 전투 전에 배치 순서를 확정하고 전투 중에는 바꿀 수 없다.
- 각 동료는 보유 스택을 기준으로 자동으로 스킬을 선택한다.
- 자동행동 우선순위는 전투 전 배치한 순서대로 적용한다.
- 스택 부족 시 대체 행동 없이 스킵한다.
- 스킵 자체에는 별도 체력/스트레스 페널티를 두지 않는다.
- 역할군 기본 구조는 딜러/탱커/서포터이며, **역할 중복 편성**을 허용한다. (예: 딜러 3 + 서포터 1)
- (MVP) 파티 최대 배치는 **4명 고정**이며, 전열/후열은 UI/타겟 규칙에서만 쓰는 개념으로 둔다.

## 동료 행동 규칙 (2026-03-15 보강)
- 미행동한 동료는 다음 턴에 보상을 받는다.
  - 해당 역할 스택 +1
  - 다음 턴 행동 순서를 맨 앞으로 당긴다.
- 예시: `1-2-3-4`에서 3번이 미행동이면 다음 턴 `3-1-2-4`
- MVP 단계에서는 복잡한 자동행동 분기보다 단순 우선순위 규칙을 먼저 구현한다.

## 선공 판정 규칙 (확정)
매 턴 시작 시 아래 값을 비교해 선공을 결정한다.

- 아군 선공 점수 = (아군 동료가 보유한 스택 합) + (이번 턴 플레이어가 사용한 카드의 스택 합)
- 적 선공 점수 = 몬스터가 보유한 스택 합

판정:
- 아군 선공 점수 > 적 선공 점수 → 아군 선공
- 아군 선공 점수 < 적 선공 점수 → 적 선공
- 동점일 때 → 랜덤으로 선공 결정

## 동점 연출 규칙 (확정)
- 동점 발생 시 시스템적으로는 랜덤 선공을 적용한다.
- 게임 내 UI/연출은 **코인 토스(앞/뒤)** 방식으로 선공 결과를 보여준다.
- 레퍼런스: **유희왕 마스터 듀얼의 코인 토스 연출**

## 스트레스 규칙 (2026-03-09 반영)
- 스트레스는 다키스트 던전의 패닉 시스템을 참고한다.
- 피격 시 입은 피해량 비례로 스트레스가 오른다.
- 동료 사망 시 추가 스트레스가 발생한다.
- 누적 스트레스는 동료 타입별 디버프로 연결한다.
