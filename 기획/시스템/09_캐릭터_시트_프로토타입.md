## 5. SO 구조 확정

### MVP 구조 (현재)

```
AllyDefinition (ScriptableObject)
├── string id
├── string displayName
├── JobClass jobClass
├── Role role
├── int sequence
├── int maxHp
├── int stressResist
├── int recruitCost
├── SkillDefinition skill1
├── SkillDefinition skill2
├── AllyNameTableSO maleNameTable
├── AllyNameTableSO femaleNameTable
├── Gender gender
└── Sprite portrait
```

```
SkillDefinition (ScriptableObject)
├── string id
├── string displayName
├── string skillGroup
├── StackType costType
├── int costAmount
├── Targeting targeting
├── EffectType effectType
├── int power
├── int aiPriority
├── StatusEffect statusEffect
├── int statusValue
└── string description
```

### 실제 구현 전환 시

```
SkillPoolSO (ScriptableObject)
├── JobClass jobClass
├── SkillDefinition[] commonSkills
└── SkillDefinition[] rareSkills

AllyDefinition (변경)
├── SkillPoolSO skillPool
└── int skillSlotCount = 2
```

## 6. 이름 생성 규칙

- ID: `ally_{jobClass}_{sequence}`
- displayName: 런타임에서 `AllyNameGenerator`가 gender 기준으로 테이블 선택 후 조합
