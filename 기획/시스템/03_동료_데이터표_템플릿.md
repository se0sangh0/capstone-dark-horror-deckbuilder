# 동료 데이터표 템플릿

> 구현용 데이터 스키마(스크립터블오브젝트)는 여기로: [데이터_테이블_v0](01_데이터_테이블_v0.md)

> 목적: 캡스톤 MVP 기준으로 동료(4인 파티, 딜러/탱커/서포터, 공용 스택 자동행동)를 설계하고 밸런싱하기 위한 실무용 정리 노트

## 사용 원칙
- MVP에서는 **전투에 바로 필요한 값**부터 확정한다.
- 성향/시너지/확장 성장 요소는 뒤로 미루고, 우선 **행동 조건 / 스택 요구치 / 역할 차이**를 잠근다.
- 동료 1명당 최소 1개 행동 정의는 반드시 필요하다.
- 표는 `동료 마스터 → 행동/스킬 → 파티 조합` 순서로 채운다.

## 1. 동료 마스터 표

### 필수 컬럼 설명
- `동료ID`: 내부 식별자 (`ALLY_001` 등)
- `이름`: 표시 이름
- `직업`: 캐스터, 오펜서, 디펜더, 어택커, 프리스트, 샤먼
- `역할`: 딜러 / 탱커 / 서포터
- `기본HP`: MVP용 핵심 스탯
- `스트레스저항`: 스트레스 시스템 반영용 최소 수치
- `기본행동ID`: 아래 행동 표와 연결되는 키
- `모집비용`: 성급 기준 고정
- `상태`: 미구현 / 초안 / 테스트중 / 확정
- `비고`: 밸런스 메모, 구현 주의사항

| ALLY_002 |     | 오펜서  | 딜러  |  150 |      5 | ACT_002 | 미구현 |     |

## 2. 행동 / 스킬 표

### 필수 컬럼 설명
- `행동ID`: 내부 식별자 (`ACT_001` 등)
- `사용동료ID`: 어떤 동료가 이 행동을 쓰는지
- `행동명`: UI 표시용
- `행동타입`: 스킬1 / 스킬2 / 디버프
- `필요스택종류`: 현재 사용하는 공용 스택 종류
- `필요스택량`: 소비 수치
- `우선순위`: 자동행동 시 먼저 판단되는 기준
- `대상규칙`: 최전열 적 / 체력 낮은 아군 / 자신 / 전체
- `부가효과`: 출혈 / 실드 / 도발 / 스트레스 회복 등

## 4. MVP에서 먼저 잠글 것

### A. 동료별 필수 확정 항목
- 역할(딜러/탱커/서포터)
- 필요 스택 종류/수량
- 행동 실패 시 처리 방식
- 모집 비용

### B. 팀 전체 기준 필수 확정 항목
- 동료 총 6종
- 역할 분배 비율
- 전열/후열 커버 가능한 조합
- 서포터/탱커 부재 조합도 돌아가는지 여부
- 카드 3종으로 필요한 스택을 안정적으로 공급 가능한지 여부

---

## 5. 나중에 붙이면 좋은 확장 컬럼

### 확장 후보
- 성향
- 성향 시너지
- 직업 시너지
- 희귀도 / 성급
- 합성 재료 여부
- 초상화 경로
- 전용 패시브
- 전투 대사 / 연출 태그

> 메모: 위 항목은 지금 넣어도 되지만, MVP 단계에서는 빈 칸으로 두거나 별도 문서에서 관리하는 편이 안전하다.

---

## 6. 빠른 점검 체크리스트

- [ ] 6명 전원 역할이 확정되었는가?
- [ ] 6명 전원 기본행동이 1개 이상 정의되었는가?
- [ ] 행동마다 필요 스택량이 적혀 있는가?
- [ ] 행동 실패 시 처리(스킵/대체행동)가 적혀 있는가?
- [ ] 전열/후열 배치 규칙이 모순 없이 들어갔는가?
- [ ] 카드 3종만으로도 실제 파티 운용이 가능해 보이는가?
- [ ] 특정 동료만 지나치게 필수 파츠가 되지 않는가?

---

## 7. 추천 작업 순서
1. 동료 6종 이름 / 역할 / 직업 확정
2. 각 동료의 기본행동 1개씩 작성
3. 스택 요구치와 대상 규칙 입력
4. 파티 2~3개 조합 만들어서 과소/과다 소비 체크
5. 그 다음에 성향/시너지 붙이기
