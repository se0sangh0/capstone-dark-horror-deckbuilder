# 카드 설계 프레임

> 구현용 데이터 스키마(스크립터블오브젝트)는 여기로: [데이터_테이블_SO_스키마](01_데이터_테이블_SO_스키마.md)

---

## 확정 원칙 (2026-03-09 / 2026-04-07 갱신)

- 카드 표현은 할리갈리식의 직관적 속성/수치 읽기 방식을 우선 채택한다.
- MVP 카드 풀은 역할 3종 × 10장씩 = **30장**으로 시작한다.
- **카드는 스택 생성 전용이다. 카드 사용으로 스킬은 발생하지 않는다.**
- 스택 수치는 카드가 아닌 **동료의 성향**이 결정한다.

---

## 카드 구조

### 카드 공통 필드
- **역할(CardType)**: Dealer / Tank / Support 3종
- **카드 이미지(cardArt)**: 역할별 3종 (미정, 역할당 1종 유력)
- ~~cardPower~~ → ❌ **미구현** (백로그: 스택 가중치 카드 / 성향·직업 시너지 확장 시 검토)

### 카드가 하는 일
```
카드 사용
    ↓
동료의 성향 범위 내에서 스택 수치 랜덤 확정 (사용 순간 생성)
    ↓
공용 스택에 반영
```

> 카드를 손패에 들고 있는 시점에는 수치가 숨겨져 있으며,
> 실제 사용 순간 동료의 성향 규칙에 따라 즉시 확정된다.
> → 매 전투마다 다른 스택 결과를 만들어내는 핵심 변수.

---

## 스택 수치 생성 규칙

- 수치는 동료의 성향(Affinity)이 결정한다. 카드 자체는 수치를 고정하지 않는다.
- 각 성향별 범위 및 생성 방식은 [동료_성향_세부기준](06_동료_성향_세부기준.md) 참조.
- 독립시행 균등확률 원칙 (2026-03-09 확정).

| 성향 | 스택 범위 (0제외) | 생성 방식 |
|---|---|---|
| 도박사 | -5 또는 +5 | 극단값 50:50 |
| 안전주의자 | -1, 1, 2 | 균등 확률(3개) |
| 기회주의자 | -2, -1, 1, 2, 3 | 균등 확률(5개) |
| 낙천가 | -3, -2, -1, 1, 2, 3, 4 | 균등 확률(7개) |

## 덱 구성 규칙

### 덱 등록 방식
- 덱은 **배치된 동료의 역할과 성향**을 바탕으로 자동 등록된다.
- **1단계 (역할 필터)**: 동료 역할에 맞는 카드만 후보군에 들어온다.
  - 예시: 탱커 동료 → 탱커 카드 10장만 후보
- **2단계 (성향 적용)**: 후보군 카드에 동료 성향의 스택 범위가 적용된다.
  - 예시: 탱커 + 도박사 동료 → 탱커 카드 10장, 사용 시 -5 또는 +5 확정
- 동료 1명당 **10장**, 파티 4인 기준 시작 덱 **40장** 고정.

### MVP 시작 덱 조건
- 시작 손패: **4장** (배치 동료 수 기준)
- 시작 덱: **40장**

---

## 손패 / 드로우 규칙

- **손패 유지량**: 배치된 동료의 수와 동일하다. (4인 배치 → 손패 4장)
- **드로우 시점**: 턴 시작 시점에 사용한 수만큼 재드로우한다.
  - 예시: 손패 4장에서 3장 사용 → 다음 턴 시작에 3장 재드로우 (1장 유지)
- **턴당 사용 제한**: 없음 (최소/최대 제한 없음)

---

## 탈진

- **탈진 조건**: 드로우 단계에서 덱에 드로우할 카드가 없는 상태
- **탈진 번 데미지**: ❌ **미구현 (1차 밸런스 단계 예정)**
- **탈진 발동 위치**: 드로우 페이즈 (턴 구조 2단계)

---

## 백로그

- `cardPower` 필드 활용: 스택 가중치 카드, 성향·직업 시너지 확장 시 검토
- 카드 런타임 동적 생성: 1장 템플릿 → 성향 적용 인스턴스 생성 방식 검토
