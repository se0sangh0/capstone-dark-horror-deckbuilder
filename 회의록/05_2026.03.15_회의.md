# 2026.03.15 회의

- 일시: 2026-03-15
- 참석자: 서상호 / 고성모 / 권성현
- 목표: 기존 안건(A1~A5)을 실제 구현 우선순위 기준으로 다시 정리하고, 이번 주 작업 범위와 담당을 잠근다.

## 논의

### 1. 코어 전투 루프 재정리
- 자동행동 구조 자체는 유지한다.
- 다만 발동 규칙이 너무 복잡하면 구현과 이해가 모두 어려워지므로, 초기 버전은 단순한 우선순위 규칙으로 정리한다.
- 미행동 처리에 대한 보상 규칙이 핵심 논점이었고, 아래 방향으로 합의가 모였다.
  - 미행동 시 해당 역할 스택 +1
  - 다음 턴 행동 순서를 미행동 동료 우선으로 재정렬
- 턴 순서 예시:
  - 기존 `1-2-3-4`
  - 3번이 미행동이면 다음 턴 `3-1-2-4`
- 자동행동 우선순위는 최종 밸런스보다 구현 가능성과 테스트 편의성을 우선한다.

### 2. 스택 / 스킬 발동 방식 논의
- 공용 스택을 카드로 생성하고, 동료가 이를 소모해 행동하는 구조는 유지한다.
- 완전 복잡한 자동 로직보다는 플레이어와 팀원이 이해하기 쉬운 단순 규칙이 우선이라는 데 공감이 형성되었다.
- MVP 단계에서는 “우선순위가 높은 스킬부터 처리” 같은 단순한 규칙 기반 구현을 우선 적용하고, 세부 로직은 QA 이후 정교화한다.

### 3. 동료 / 스킬 / 조합 구조
- 동료 모집 시 스킬은 랜덤 부여 구조를 유지한다.
- 단, 중복 스킬이 과도하게 겹치지 않도록 중복 방지 규칙을 둔다.
- 덱 자유도는 지나치게 넓히지 않고, 초기 버전은 통제된 랜덤성과 조합 차별화에 집중한다.

### 4. 성향 / 시너지 / 콘텐츠 축소 여부
- 성향과 직업 시너지는 아이디어로 남아 있으나, 현재 단계에서 전부 구현하기에는 범위가 과하다는 공감대가 있었다.
- 따라서 MVP에서는 성향 시스템을 핵심 축에서 내리고, 시너지도 기본 구조 완성 이후 검토하기로 했다.
- 단, 완전 폐기보다 “대표 시너지 1개 정도만 데모성으로 남길 수 있는지”는 열어둔다.

### 5. MVP 범위 재확인
- 이번 버전의 목표는 예쁘고 풍부한 게임이 아니라, 실제로 한 판 돌아가는 프로토타입이다.
- 우선 구현 범위는 아래와 같이 재정리되었다.
  - 맵 노드
  - 전투
  - 플레이어 화면 / UI
  - 기본 보스 흐름
  - 기초 시스템
- 세부 콘텐츠, 복잡한 시너지, 추가 성향 설계, 과한 확장 기능은 후순위로 미룬다.

### 6. UI / 구현 분담 방향
- UI는 이번 주 중요한 산출물로 간주한다.
- 상호는 피그마를 익히면서 기준 화면 설계를 맡고, 해상도는 우선 FHD(1920x1080) 기준으로 작업한다.
- 성모는 UI 구현/시각화 및 관련 검토에 참여한다.
- 성현은 유니티 프로젝트와 에셋, 구현 연결을 담당한다.

### 7. 일정 / 제출 관련
- 이번 주에 할 작업 내용은 잠갔지만, 최종 제출 마감은 교수님 일정 확인 후 다시 확정하기로 했다.
- 즉, 현재 회의는 제출일 고정보다는 작업 방향과 역할 분배를 확정하는 데 목적이 있었다.

## 결정
- 자동행동 구조는 유지한다.
- 미행동 보상 규칙을 채택한다.
  - 미행동 시 해당 역할 스택 +1
  - 다음 턴 행동 순서를 앞으로 당긴다.
- MVP에서는 복잡한 자동 로직보다 단순 우선순위 규칙을 우선 구현한다.
- 동료 모집 시 스킬 랜덤 구조는 유지하되, 중복 방지 규칙을 둔다.
- 성향 시스템은 MVP 핵심축에서 후순위로 내린다.
- 시너지는 전면 구현 대신, 필요 시 대표 사례 1개 수준만 검토한다.
- 이번 주 우선순위는 콘텐츠보다 시스템 / 전투 / UI 구조다.
- MVP는 ‘돌아가는 플레이 가능 프로토타입’을 목표로 한다.
- UI 기준 해상도는 우선 FHD(1920x1080)로 잡는다.
- 최종 마감은 교수님 일정 확인 후 재확정한다.
