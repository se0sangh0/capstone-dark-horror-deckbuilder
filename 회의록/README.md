# 회의록 폴더 규칙

## 목적
- 팀 회의 내용을 날짜 기준으로 누적 보관한다.
- 회의에서 결정된 내용이 허브/설계 문서와 분리되지 않도록, 회의 직후 바로 문서 반영 흐름을 만든다.

## 폴더 사용 규칙
- 모든 회의 관련 문서는 `회의록/` 폴더 아래에 저장한다.
- 회의 날짜별로 최소 1개의 회의록 파일을 남긴다.
- 필요하면 같은 날짜에 결정사항 / 액션아이템 문서를 추가로 둔다.

## 파일명 규칙
- 기본 회의록:
  - `YYYY-MM-DD_회의.md`
- 결정사항 요약본(선택):
  - `YYYY-MM-DD_결정사항.md`
- 액션아이템 분리본이 필요하면:
  - `YYYY-MM-DD_액션아이템.md`
- 기존 번호 체계가 필요한 경우에도, 장기적으로는 위 날짜 중심 규칙을 우선한다.

## 작성 원칙
- 반드시 포함:
  - 일시
  - 참석자
  - 회의 목표
  - 논의 내용
  - 결정된 것
  - 액션아이템
- 회의에서 확정되지 않은 내용은 `미결정` 또는 `추후 논의`로 분리한다.

## 회의 직후 운영 규칙
1. `회의록/`에 회의록 저장
2. `00_프로젝트_허브.md` 링크 갱신
3. 관련 설계 문서(코어루프/MVP/역할분담/UI 등) 반영
4. Git 커밋 및 원격 반영

## 권장 커밋 예시
- `docs(capstone): add YYYY-MM-DD meeting notes`
- `docs(capstone): sync meeting decisions into project hub`
