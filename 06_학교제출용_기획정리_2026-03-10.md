# 학교 제출용 기획 정리 (2026-03-10)

## 문서 목적
학교 제출 양식에 맞춰 확정/가안 상태를 구분해 기획 핵심을 정리한다.

---

## 1) 제출본 반영 요약
- 게임명(가칭): **괴이탐사국: 다크 호러 덱빌딩 오토배틀러**
- 팀명: **578**
- 팀장: **서상호**
- 과제 목표: 덱빌딩+오토배틀러 하이브리드 핵심 루프를 MVP로 구현하고 플레이 테스트로 재미 검증
- 핵심 기술: 공용 스택 기반 전투 로직, `GameState` 상태 관리, 패널 전환형 UI 아키텍처
- 개발 범위: 카드/동료/노드/스트레스 MVP 구현 후 콘텐츠 확장

---

## 2) 게임 개요(제출본 기준)
1. 게임 제목: 괴이탐사국: 다크 호러 덱빌딩 오토배틀러 (가칭)
2. 플랫폼: PC(Windows) 우선, 추후 모바일 환경 지원 검토
3. 기획 의도: 로그라이크 탐험 + 덱빌딩·오토배틀러 결합 전략형 전투 경험 구현
4. 시점: UI 중심 탑뷰형 진행
5. 장르: 다크 호러 로그라이크 / 덱빌딩 / 오토배틀러
6. 재미 요소: 공용 스택 설계, 동료 배치 순서 자동행동, 스트레스 시스템
7. 조작법: 마우스 중심 + 단축키 보조
8. 클리어 조건: MVP 테스트 시나리오 클리어(향후 정식 엔딩 확장)
9. 주요 오브젝트: 카드, 동료(딜러/탱커/서포터), 노드, 적, 재화(영혼석/마석), 스트레스

---

## 3) 확정/가안 구분
### 확정
- MVP 우선 개발 원칙
- 고정 HUD + 중앙 패널 전환 UI 구조
- 4인 고정 파티/전투 중 교체 불가
- 공용 스택 기반 전투 흐름

### 가안(추후 확정)
- 모바일 지원 시점/범위
- 엔딩 분기 세부 조건
- 콘텐츠 확장 우선순위(카드/직업군/이벤트 비중)
