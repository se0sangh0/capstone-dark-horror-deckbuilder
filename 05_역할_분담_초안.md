# 역할 분담 초안

## 목적
- 2026년 3월 9일 회의에서 잠근 규칙을 바로 프로토타입 작업으로 연결
- 3인 팀 기준으로 역할과 산출물을 고정해 바로 작업에 들어갈 수 있게 정리

## 현재 팀 구성
- 팀장: 서상호
- 팀원: 고성모
- 팀원: 권성현

## 3인 운영 원칙
- 역할 축은 `시스템 / 프로토타입 / 콘텐츠`로 잡고, 문서는 별도 1인 전담이 아니라 각 담당 산출물과 함께 관리한다.
- 서상호가 팀장으로서 일정 확인과 문서 최종 취합을 맡고, 테스트 로그와 발표 자료 검수는 전원이 함께 진행한다.
- 각 인원은 `주담당 1개 + 보조 참여 1개`만 가진다.
- 아직 협의가 끝나지 않은 항목은 템플릿 수준으로 유지하고, 합의된 항목만 실제 수치나 문안으로 잠근다.

## 권장 역할 배정안

### 서상호(팀장) - 시스템 / 기획 정리 / UI 설계
- 책임 범위
  - 손패 리셋 규칙
  - 공용 스택 누적 / 소모
  - 미행동 보상 규칙
  - UI 구조와 화면 배치 기준
- 이번 주 산출물
  - 전투 UI 초안(Figma)
  - 시스템 우선 화면 배치 기준안
  - 허브/회의록 반영본 최종 취합
- 보조 참여
  - 발표용 1페이지 요약 정리
  - 테스트 체크리스트 취합

### 고성모 - UI / 프로토타입 보조
- 책임 범위
  - UI 구성 시각화 또는 초안 구현
  - 전투 UI 구조 검토
  - 필요 시 시너지 아이디어 보조 탐색
- 이번 주 산출물
  - UI 구성안 초안
  - 시각화/와이어프레임 또는 초기 구현 결과물
- 보조 참여
  - 내부 테스트 피드백 정리

### 권성현 - 구현 / 유니티 프로젝트 연결
- 책임 범위
  - 유니티 프로젝트 세팅
  - 에셋 연결
  - 전투/시스템 구현 연결
  - PR 시도 및 구현 블로커 공유
- 이번 주 산출물
  - 유니티 프로젝트 준비본
  - 구현 연결 상태 공유
  - PR 또는 작업 브랜치 기준 진행 상황 정리

## 공통 문서 / 테스트 운영
- 허브, 회의록, 테스트 로그는 해당 작업 담당자가 먼저 갱신한다.
- 서상호가 최종 취합본을 정리하고 다음 회의 안건을 묶는다.
- 발표용 요약본은 내부 테스트 이후 전원 검수로 확정한다.

## 1달 발표 대비 업무 진행 가안 (회의용)

### 운영 원칙
- 상호가 규칙을 잠그면 해당 주차에는 규칙 변경을 최소화한다.
- 구현은 `권성현(핵심 기능)` + `고성모(통합/테스트)` 2트랙으로 진행한다.
- 매주 결과물은 `문서 1개 + 플레이 가능한 빌드 1개 + 테스트 로그`를 남긴다.

### 역할 재정의(실행 중심)

#### 서상호 (팀장 / 기획 / 문서)
- 규칙 확정 및 우선순위 결정
- 문서 싱크 최종 책임
- 발표 스토리라인 및 회의 진행

#### 권성현 (핵심 구현)
- 전투 핵심 로직(카드-스택-자동행동)
- 용병소 교체 핵심 동작 구현
- 전투 시스템 코드 품질 책임

#### 고성모 (통합 / 파이프라인 / 테스트)
- 전투-노드-보스 씬 연결 및 플레이 루프 완주 가능 상태 유지
- 확정 수치 반영(손패4/덱4/4인고정/배치순서 우선)
- 데일리 빌드/테스트 로그 관리 및 버그 우선순위 정리

### 주간 운영 루틴(반복)

#### 월요일: 주간 계획 잠금(30~45분)
- 이번 주 필수 결과물 1~2개 확정
- 담당자/마감시각/검증기준 명시
- 범위 밖 아이디어는 백로그로 이동

#### 화~목: 개발 집중
- 권성현: 핵심 기능 구현
- 고성모: 연결/데이터 반영/테스트
- 서상호: 문서 갱신/의사결정/블로커 해소

#### 금요일: 통합 테스트
- 최소 2회 완주 테스트
- 버그를 치명/중간/낮음으로 분류
- 다음 수정 우선순위 잠금

#### 일요일: 주간 리뷰
- 완료/미완료 정리
- 다음 주 필수 목표 선확정

### 데일리 스탠드업(10분)
- 어제 한 일
- 오늘 할 일
- 막힌 점(블로커)

### 첫 주 실행 과제(가안)
- 서상호: 교체 규칙 상세(타이밍/비용/초기화) 문서 잠금
- 권성현: 배치 순서 기반 자동행동 구현
- 고성모: 전투→용병소→교회→보스 루프 연결 + 테스트 체크리스트 작성
