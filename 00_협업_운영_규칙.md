# 협업 운영 규칙 (GitHub + Obsidian)

## 1) 브랜치 규칙
- `main`: 배포/공유 기준 브랜치 (직접 푸시 지양)
- 작업 브랜치 네이밍:
  - `docs/<topic>` (문서)
  - `feat/<topic>` (기능)
  - `fix/<topic>` (수정)

## 2) PR 규칙 (최소)
- PR 단위는 작게: 한 주제 한 PR
- 제목 형식: `[docs] UI 규정 반영` / `[feat] 용병소 패널 초안`
- 반드시 포함:
  - 변경 이유
  - 변경 파일 목록
  - 영향 범위(어떤 문서/시스템)

## 3) Obsidian 협업 규칙
- 모든 팀원은 **동일 Vault 루트**를 열 것:
  - `캡스톤_다크호러_덱빌딩오토배틀러`
- 구조 변경 시 먼저 `00_프로젝트_허브.md` 업데이트
- 모든 회의 문서는 `회의록/` 폴더 아래에 저장한다.
- 회의록 파일명은 날짜 중심으로 관리한다.
  - 기본: `YYYY-MM-DD_회의.md`
  - 필요 시: `YYYY-MM-DD_결정사항.md`, `YYYY-MM-DD_액션아이템.md`
- 디스코드는 회의 아카이브 저장소로 쓰지 않는다.
  - 디스코드: 실시간 논의 / 임시 로그 / 회의 재구성용 원본
  - Obsidian + GitHub: 최종 회의록 / 문서 반영 / 버전 관리 기준
- 회의 후 10분 내:
  1) `회의록/`에 회의록 저장
  2) 관련 문서 링크 갱신
  3) 허브/설계 문서 반영
  4) GitHub 기준으로 커밋/푸시

## 4) 충돌 방지 규칙
- 기기별 UI 파일은 추적 제외(`.obsidian/workspace*.json`)
- 같은 파일 동시 편집 피하기
- 큰 문서 수정 전 "내가 이 파일 수정 시작" 공유

## 5) 리뷰 규칙 (가볍게)
- 문서 PR은 최소 1인 확인 후 머지
- 수치 변경(밸런스)은 변경 근거 1줄 필수

## 6) 커밋 메시지 컨벤션
- `docs:` 문서 변경
- `feat:` 기능 추가
- `fix:` 버그/오탈자/링크 수정
- `chore:` 구조 정리/설정 변경

예시:
- `docs: 회의 안건 반영 및 용병소 규칙 명문화`
- `feat: 패널 라우터 상태 전환 초안 추가`
