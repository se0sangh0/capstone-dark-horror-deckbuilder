# 캡스톤 — 다크 호러 덱빌딩 오토배틀러

팀 문서/기획 협업용 Obsidian Vault + GitHub 저장소입니다.

## 저장소 목적
- 회의 결정사항과 설계 문서를 단일 소스로 관리
- Obsidian에서 빠르게 작성하고 Git으로 안정적으로 협업
- 구현용 Unity 프로젝트와 기획 문서를 한 저장소에서 함께 관리

## 시작하기
1. 저장소 클론
2. Obsidian에서 **이 폴더 자체**를 Vault로 열기
   - `캡스톤_다크호러_덱빌딩오토배틀러`
3. `00_프로젝트_허브.md`부터 읽고 현재 기준 문서를 확인
4. 작은 문서 작업은 바로 커밋해도 되고, 큰 작업은 브랜치 분리 권장

## 문서 구조
- `00_프로젝트_허브.md` : 최상위 허브(링크 인덱스)
- `기획/` : 게임 기획/명세(코어/시스템/서사/MVP)
- `디자인_UI/` : UI/디자인 가이드(Figma/와이어프레임/컬러 등)
- `운영/` : 협업 규칙/프로젝트 관리(로드맵/역할 등)
- `주간_체크리스트/` : 주차별 운영 체크리스트(+참고 체크시트)
- `회의록/` : 회의 기록
- `제출용/` : 제출 초안/양식 매핑/제출 파일(xlsx 포함)
- `templates/` : 템플릿
- `archive/` : 병합되었거나 보관용으로 남긴 문서

## 협업/작성 규칙(요약)
- 상세 규칙은 단일 소스: `운영/00_협업_운영_규칙.md`
- 최소 원칙 3개만 기억하면 됨:
  1) 구조/파일 이동 후에는 `00_프로젝트_허브.md` 링크부터 갱신
  2) 회의 후 10분 내 회의록 저장 + 관련 문서 반영
  3) 큰 변경(구조 개편/대량 수정/구현 리스크)은 브랜치 권장(작은 문서 수정은 main 직커밋 OK)

## Unity 파일 정책
이 저장소는 **필요 소스만 추적**합니다.

### 추적(올라감)
- `Assets/`
- `Packages/`
- `ProjectSettings/`
- 기획/문서(`*.md`)

### 비추적(안 올라감)
- `Library/`, `Temp/`, `Obj/`, `Build/`, `Logs/`, `UserSettings/`
- IDE 생성 파일(`.vs`, `.idea`, `.vscode`, `*.csproj`, `*.sln` 등)
- Syncthing 로컬 아티팩트(`.stfolder`, `.stversions/` 등)
