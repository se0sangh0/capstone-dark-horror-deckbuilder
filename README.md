# 캡스톤 — 다크 호러 덱빌딩 오토배틀러

팀 문서/기획 협업용 Obsidian Vault + GitHub 저장소입니다.

## 저장소 목적
- 회의 결정사항과 설계 문서를 단일 소스로 관리
- GitHub PR 기반으로 변경 이력/리뷰를 남김
- Obsidian에서 빠르게 작성하고 Git으로 안정적으로 협업

## 시작하기
1. 저장소 클론
2. Obsidian에서 **이 폴더 자체**를 Vault로 열기
   - `캡스톤_다크호러_덱빌딩오토배틀러`
3. 작업은 브랜치에서 진행 후 PR 생성

## 문서 구조
- `00_*` : 허브/운영 규칙/협업 규정
- `01_*` : 코어 컨셉/코어루프
- `02_*` : 전투/카드/스트레스 시스템
- `03_*` : 동료/맵/직업군
- `04_*` : MVP 범위/테스트
- `05_*` : 회의/로드맵/일정
- `99_*` : 템플릿

## 작성 규칙 (요약)
1. **파일명은 유지**하고 문서 내부를 수정한다. (이름 변경은 합의 후)
2. 구조 변경 시 먼저 `00_프로젝트_허브.md` 링크를 갱신한다.
3. 수치 변경은 변경 이유를 최소 1줄 남긴다.
4. 회의 후 10분 내 회의록 + 관련 문서 링크를 갱신한다.
5. 큰 변경 전에는 팀 채널에 “수정 시작 파일”을 공유한다.

## Git 협업 규칙
- `main` 브랜치는 보호됨 (PR + 1 approve)
- 브랜치 네이밍:
  - `docs/<topic>`
  - `feat/<topic>`
  - `fix/<topic>`
- 커밋 컨벤션:
  - `docs:` 문서
  - `feat:` 기능
  - `fix:` 수정
  - `chore:` 정리/설정

## Unity 파일 정책
이 저장소는 **필요 소스만 추적**합니다.

### 추적(올라감)
- `Assets/`
- `Packages/`
- `ProjectSettings/`
- 기획/문서(`*.md`)

### 비추적(안 올라감)
- `Library/`, `Temp/`, `Obj/`, `Build/`, `Logs/`, `UserSettings/`
- IDE 생성 파일(`.vs`, `.idea`, `.vscode`, `*.csproj`, `*.sln` 등)
- Syncthing 로컬 아티팩트(`.stfolder`, `.stversions/` 등)
