# 전투 UI 와이어프레임 가이드

## 문서 목적
- Figma에서 전투 화면 1차 시안을 빠르게 만들 수 있도록 영역 구조를 정의한다.
- 무엇을 어디에 배치해야 하는지 우선순위를 통일한다.
- 장식보다 정보 전달과 플레이 판단이 먼저 보이도록 만든다.

---

## 1) 기준 해상도
- 기본 프레임: `1920 x 1080`
- 우선 제작 대상: 가로형 PC 화면 1안

---

## 2) 전체 구조
전투 화면은 크게 5개 블록으로 나눈다.

1. 상단 바
2. 좌측 상태 패널
3. 중앙 전투 메인 필드
4. 우측 적/전황 패널
5. 하단 플레이어 입력 영역

추천 인상:
- 좌측 = 파티와 자원 상태를 읽는 계기판
- 중앙 = 실제 전투가 벌어지는 메인 무대
- 우측 = 적/턴/위험도 확인 영역
- 하단 = 플레이어가 직접 만지는 카드/행동 영역

---

## 3) 레이아웃 초안

### A. 상단 바 (전역 HUD)
포함 요소:
- 현재 라운드 / 턴
- 재화(영혼석, 마석)
- 현재 노드 또는 전투명
- 설정 / 로그 / 도움말 버튼

목적:
- 지금 어떤 전투인지, 어느 정도 진행됐는지 즉시 인지

주의:
- 상단 바는 얇고 길게 유지
- 정보는 많아도 되지만 시선을 뺏으면 안 됨

---

### B. 좌측 상태 패널 (파티 + 시스템 상태)
포함 요소:
- 파티 슬롯 4인
- 각 동료의 역할 / 체력 / 스트레스 / 상태이상
- 행동 순서 표시
- 역할 스택 또는 특수 자원 요약
- 덱 수 / 버림패 / 현재 스택 요약

목적:
- 내 편 상태를 한 번에 보는 영역

구성 권장:
- 위: 파티 요약
- 중간: 행동 순서 / 역할 자원
- 아래: 덱/스택/스트레스 요약

포인트:
- 동료가 다음 턴에 먼저 행동하는지 여부가 분명히 드러나야 함
- “미행동 보상” 같은 핵심 규칙은 이 패널에서 읽혀야 함

---

### C. 중앙 전투 메인 필드
포함 요소:
- 동료 배치 위치
- 적 배치 위치
- 공격/효과 발생 위치
- 턴 진행의 중심 시각 요소

목적:
- 전투의 감정과 상황을 보여주는 메인 무대

구성 권장:
- 좌측 하단 쪽: 아군
- 우측 상단 또는 우측 중앙: 적군
- 가운데: 충돌/스택 소비/행동 연출 공간

주의:
- 중앙은 예쁘게 만드는 것도 중요하지만, MVP에서는 너무 복잡한 배경보다 전투 대상 배치가 잘 보이는 게 우선
- 연출보다 배치 판독성이 중요

---

### D. 우측 적/전황 패널
포함 요소:
- 적 목록 또는 현재 타깃 정보
- 적 체력 / 상태 / 특성
- 위험도 표시
- 전투 로그 미니 요약

목적:
- 지금 어떤 적이 위협적인지 빠르게 읽게 함

포인트:
- 보스전에서는 보스 전용 정보 영역으로 확장 가능
- 일반 전투에서는 간단한 리스트형도 가능

---

### E. 하단 플레이어 입력 영역
포함 요소:
- 손패 카드 나열
- 카드 상세 툴팁 자리
- 현재 생성 가능한 스택 예고
- 턴 종료 / 확인 버튼

목적:
- 플레이어가 직접 의사결정하는 중심 영역

구성 권장:
- 하단 중앙에 카드 4~6장 기준 배치
- 선택 카드 강조 테두리
- 카드 사용 시 공용 스택 변화가 근처에서 바로 보이면 좋음

주의:
- 카드 장식보다 수치와 속성 가독성이 우선
- 클릭/선택 가능한 영역처럼 보여야 함

---

## 4) 최소 와이어프레임 블록 구성
1차 시안에서는 아래 직사각형만 먼저 잡아도 충분하다.

- Frame / 1920x1080
  - TopBar
  - LeftStatusPanel
  - CenterBattleField
  - RightEnemyPanel
  - BottomHandArea
  - PopupLayer(옵션)

---

## 5) 정보 우선순위 규칙
플레이어가 먼저 봐야 하는 것 순서:
1. 지금 위험한가?
2. 내가 낼 카드가 뭔가?
3. 공용 스택이 얼마나 생기거나 부족한가?
4. 다음에 누가 행동하는가?
5. 적이 어떤 상태인가?

이 순서를 거스르는 배치는 피한다.

---

## 6) 피그마 컴포넌트 초안 추천
처음부터 아래 컴포넌트만 만들면 된다.

- `Panel / Base`
- `Button / Primary`
- `Button / Danger`
- `Card / Hand`
- `Card / Selected`
- `PartySlot / Normal`
- `PartySlot / Active`
- `EnemySlot / Normal`
- `Badge / Stress`
- `Badge / Stack`
- `TopBar / Resource`

---

## 7) 와이어 단계에서 굳이 안 해도 되는 것
- 복잡한 텍스처
- 캐릭터 일러스트 확정
- 완성형 아이콘 세트
- 화려한 전투 이펙트
- 세부 애니메이션

---

## 8) 최종 체크리스트
- [ ] 화면을 보자마자 전투 화면인지 알 수 있는가?
- [ ] 손패와 공용 스택 위치가 직관적인가?
- [ ] 동료 상태와 행동 순서가 바로 보이는가?
- [ ] 적 정보가 너무 멀거나 작지 않은가?
- [ ] 로그/HUD가 메인 플레이 영역을 방해하지 않는가?

---
